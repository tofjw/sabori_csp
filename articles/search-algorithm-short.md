# 「論理」ではなく「傾向」で無駄な探索を止める制約ソルバー

*自作の FlatZinc/CP ソルバー sabori_csp の短い紹介です。数表・方法論・ソース参照を含む全文は末尾リンクから。*

sabori_csp は標準的な骨格——バックトラッキング + 制約伝播 + リスタート + activity ベースの変数選択 + NoGood 学習——を持つ制約ソルバーです。個性は、**どうやって無駄な探索を止めるか**にあります。LCG（Lazy Clause Generation、Chuffed 系）と対比すると一番はっきりします。

> **LCG は「論理」で無駄を止める。矛盾から健全な節を学習し、二度と通らない領域を演繹的に枝刈りする。sabori_csp は「傾向」で止める。同じ矛盾情報を主に activity に流し込み、変数選択というヒューリスティクスの傾向を、悪い領域から逸らす。**

意図的に軽量です——**LCG なし、含意グラフ解析なし、伝播器ごとの explanation なし**。代わりに、安価で自己調整するヒューリスティクスに寄せています。以下、実際に何をしていて、（全部 A/B 計測したので）どこが効いているかを並べます。

## 効いている手法

各機能を環境変数のトグルに分け、MiniZinc Challenge で複数シード・目的値判定で計測しました（数表と方法論は全文に）。利きが確認できたのは次の3つです。

- **弱い NoGood 学習が、activity 経由で効く。** 矛盾時、sabori は決定トレイルをそのまま1つの NoGood にするだけ——1UIP や LCG よりずっと弱い。なのに切ると全シードで一貫して悪化します。面白いのは内訳で、効果は**枝刈りより activity 寄与が主**。NoGood 由来の activity bump だけを止めると、純粋な枝刈りだけを止めるより大きく悪化する。学習した矛盾は、**activity を強め探索の傾向を作り替える**ことで主に効いている、ということです（＝タイトルの「論理でなく傾向」を計測で裏付けた形）。
- **変数選択の混合比を回すバンディット。** 「activity 優先 vs MRV 優先」を5本腕のマルチアームド・バンディットにして、リスタートごとに抽選し直す。最良の固定ヒューリスティクスには勝てないが、**問題ごとに最悪の固定を自動で避ける**。事前に正解を選べない状況での保険です。
- **one-hot チャネル集約の presolve。** FlatZinc は整数を one-hot のブール群に展開しがちで、それが大量の小さな reif 制約として降ってくる。sabori はこれを1つのチャネル制約に融合する。平均では控えめな勝ち、しかし問題によっては探索量を劇的に減らします。

つまりエンジンの実体は、**activity を育てる弱い学習器とバンディット、そしてモデルを軽くする presolve**。安価な部品で、傾向によって舵を取っています。

## 効かなかったもの

隠さず出します——ソルバーにとって「何が冗長か」は「何が効くか」と同じくらい有用なので。

- **NoGood-Bloom 重なりによる変数選択タイブレーク。** ほぼ no-op で、大半の問題では何も変わらない。activity がすでにその情報を運んでいるので、タイブレークに出番が来ない。
- **制約ごとの構造的な責任配分。** Circuit の `occupier_`、AllDifferent の値プールなどから、矛盾の真犯人を変数単位で指す——健全性を要らない「poor man's explanation」（ヒューリスティクス専用なので不健全でよい）。気に入っていた仕掛けですが、generic な版（動いた変数を一律に責めるだけ）に対して測定可能な利得なし。シードで勝敗が反転する。構造特化は何も買っていませんでした → future work。
- **擬似勾配の値選択ヒント。** 平均では負け。ただし問題で割れ、設計・割当系では効き、資源結合の強いスケジューリングでは裏目に出る。常時 ON は外すべきだが、並列ポートフォリオの一本としては価値がある。

一貫したパターンがあり、これが本当の教訓だと思います——**activity を育てる土台は効く。その活きた activity の上に積んだ賢さは、たいてい効かない。activity がすでに支配的な信号だからです。** これが「論理でなく傾向」の言い換えとも符合します。sabori も演繹的な枝刈りはしている。だが実際に無駄を切っているのは、主に傾向の側です。
