# 「論理」ではなく「傾向」で無駄な探索を止める制約ソルバー

*自作の FlatZinc/CP ソルバー sabori_csp の短い紹介です。数表・方法論・ソース参照を含む全文は末尾リンクから。*

sabori_csp は標準的な骨格——バックトラッキング + 制約伝播 + リスタート + activity ベースの変数選択 + NoGood 学習——を持つ制約ソルバーです。個性は、**どうやって無駄な探索を止めるか**にあります。LCG（Lazy Clause Generation、Chuffed 系）と対比すると一番はっきりします。

> **LCG は「論理」で無駄を止める。矛盾から健全な節を学習し、二度と通らない領域を演繹的に枝刈りする。sabori_csp は「傾向」で止める。同じ矛盾情報を、変数選択のヒューリスティクス——最近揉めた変数へ寄せる conflict-directed な傾向——に流し込み、悪い領域から逸らす。**

意図的に軽量です——**LCG なし、含意グラフ解析なし、伝播器ごとの explanation なし**。代わりに、安価で自己調整するヒューリスティクスに寄せています。以下、実際に何をしていて、（全部 A/B 計測したので）どこが効いているかを並べます。

## 効いている手法

各機能を環境変数のトグルに分け、MiniZinc Challenge で複数シード・目的値判定で計測しました（数表と方法論は全文に）。

- **変数選択そのものが効く土台で、2つが探索フェーズを分担している。** バックトラック直後の最初の1手は、**conflict-directed な第1基準（Last Conflict に着想を得た拡張）**が「直近で揉めた変数」を最優先で選ぶ——単一の機構としてはこれが一番効く。そこから木を順調に降りる間は、**activity が選択を駆動**する。temporal が「再開地点」を、activity が「降下」を決める分担です。activity の効きは temporal にマスクされて小さく見えますが、temporal を外すと真価が出ます（全文で定量）。
- **その activity を供給するのが、弱い NoGood 学習。** 矛盾時、決定トレイルをそのまま1つの NoGood にするだけ——1UIP や LCG よりずっと弱い。なのに切ると全シードで悪化。効果は枝刈りより **activity 寄与が主**で、学習した矛盾は activity を強めて傾向を作り替えることで効いている（＝タイトルの「論理でなく傾向」を計測で裏付けた形）。
- **変数選択（activity 軸）の混合比を回すバンディット。** 「activity 優先 vs MRV 優先」をリスタートごとに抽選し直す。最良の固定には勝てないが、**問題ごとに最悪の固定を自動で避ける**保険。
- **one-hot チャネル集約の presolve。** FlatZinc は整数を one-hot のブール群に展開しがちで、それが大量の小さな reif 制約として降ってくる。sabori はこれを1つのチャネル制約に融合する。平均では控えめな勝ち、しかし問題によっては探索量を劇的に減らします。

つまりエンジンの実体は、**conflict-directed な変数選択（Last Conflict 系の第1基準 + activity 駆動の降下）と、それを支える弱い NoGood・バンディット、モデルを軽くする presolve**。安価な部品で、傾向によって舵を取っています。

## 効かなかったもの

隠さず出します——ソルバーにとって「何が冗長か」は「何が効くか」と同じくらい有用なので。

- **NoGood-Bloom 重なりによる変数選択タイブレーク。** ほぼ no-op で、大半の問題では何も変わらない。activity がすでにその情報を運んでいるので、タイブレークに出番が来ない。
- **制約ごとの構造的な責任配分。** Circuit の `occupier_`、AllDifferent の値プールなどから、矛盾の真犯人を変数単位で指す——健全性を要らない「poor man's explanation」（ヒューリスティクス専用なので不健全でよい）。気に入っていた仕掛けですが、generic な版（動いた変数を一律に責めるだけ）に対して測定可能な利得なし。シードで勝敗が反転する。構造特化は何も買っていませんでした → future work。
- **擬似勾配の値選択ヒント。** 平均では負け。ただし問題で割れ、設計・割当系では効き、資源結合の強いスケジューリングでは裏目に出る。常時 ON は外すべきだが、並列ポートフォリオの一本としては価値がある。

一貫したパターンがあり、これが本当の教訓だと思います——**conflict-directed な変数選択（第1基準の Last Conflict + 降下を駆動する activity）という土台は効く。その上に積んだ賢さ（Bloom・構造特化・勾配）は、たいてい効かない。土台がすでに探索を決めているからです。** これが「論理でなく傾向」の言い換えとも符合します。sabori も演繹的な枝刈りはしている。だが実際に無駄を切っているのは、主に傾向の側です。
