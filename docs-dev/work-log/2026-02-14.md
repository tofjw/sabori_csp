## 2026-02-14 Domain Splitting (Bisection) + 拡張 NoGood 学習の実装

**実施内容**
- `Literal` 型に `Type` enum (`Eq`, `Leq`, `Geq`) を追加し、`is_satisfied()` / `negate()` メソッドを実装
- NoGood Watch を Eq/Leq/Geq の3つのマップに分割（`ng_eq_watches_`, `ng_leq_watches_`, `ng_geq_watches_`）
- `propagate_nogood()` を全リテラル型対応に書き換え（Unit propagation: Leq→set_min, Geq→set_max）
- `propagate_bound_nogoods()` を新規実装し、SetMin/SetMax/instantiate 時にトリガー
- `run_search()` に Bisect モード追加（`x ≤ mid` / `x > mid` の二分割分岐、O(log d) に削減）
- ヒント解に基づく分岐順序最適化（`right_first` フラグ）
- `-b N` CLI オプション追加（デフォルト 8、0=無効）
- `SolverStats` に `bisect_count` / `enumerate_count` 追加

**設計判断・理由**
- Bisection threshold のデフォルトを 8 に設定: 小さなドメインでは値列挙の方が効率的（穴あきドメインへの対応含む）
- Watch を3分割した理由: Eq は (var, value) ペアで O(1) ルックアップ、Leq/Geq は var ごとのリストで境界変更時にスキャン
- Bisect 時の decision_trail には Leq/Geq リテラルを記録し、NoGood 学習に自然に統合
