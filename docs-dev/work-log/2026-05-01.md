## 2026-05-01 16:10 真の root cause 発見と修正

**実施内容**
- ユーザーから「全部の値が即時 fail なら正しい UNSAT のはず」との指摘。完備性仮説を棄却し、伝播の正しさを再調査

  ```cpp
  if (y_min < 0) update_z(y_min);  // y が負のときのみ
  if (y_max > 0) update_z(y_max);
  if (y_min <= -1 && y_max >= -1) update_z(-1);
  if (y_min <= 1 && y_max >= 1) update_z(1);
  ```

  y が完全に正 ([814, 8140] 等) のとき、`y_min` は端点として考慮されない

**設計判断・理由**

**他制約の点検結果** (同種バグなし)
- **IntTimes**: forward は 4 端点積 `{x_min*y_min, x_min*y_max, x_max*y_min, x_max*y_max}` の min/max。symbolic にどんな符号にも対応できる定石。backward も `div_ceil/div_floor` の 4 端点。`y` が 0 を跨ぐ場合のみ backward をスキップする保守的ガード

- 一般的な原理: `f(x, y)` の極値が y の端点で発生する単調関数では、両端点を必ず候補にする必要がある
