## 2026-03-25 init_activity 改善試行・nogood lifetime 調査

**実施内容**
- init_activity の log(domain_size) ベース化を試行
  - 基底クラスに `log(d)` デフォルト追加、linear制約で `log(d) × log1p(|coeff|)` に変更
  - reif/imp 制約は b のみに activity 設定、non-reif は全変数
  - 最大値 1.0 へのスケーリング追加
  - ベンチマーク結果わずかに悪化 → 全て revert
- reif/imp 制約の init_activity を no-op 化を試行
  - ベンチマーク結果変化なし → revert
- nogood_inactive_restart_limit_ の最適値調査
  - 10, 30, 50, 100, 200 を試行
  - 劇的な差なし → 10（デフォルト）に戻す

**設計判断・理由**
- init_activity: 情報理論的に `log(d)` は妥当だが、実際のベンチマークではアドホックな係数比率と同等以下。探索中の bump が初期値に埋もれる問題あり（スケーリングで対処したが改善せず）
- reif の init_activity: b が未確定のとき他の変数を決めても b が変わるだけで束縛が生じないため、activity 0 が理論的に正しいが、ベンチマーク上は差なし
- nogood inactive limit: 10〜200 の範囲で大きな差なし。長く保持しても発火率は低く、メモリ増加のみ
