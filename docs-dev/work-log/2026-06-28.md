## 2026-06-28 improvement probe の ablation（§7 の未計測機構）

**実施内容**
- 指摘: §7 には独立した2機構がある——勾配ヒント（値順序バイアス）と improvement probe（`run_improvement_probe`、ベスト目的値側から ~5% 改善を狙う軽量サブ探索）。`SABORI_GRADIENT` で測ったのは勾配ヒントの方で、**probe を走らせるか否かは未計測**だった。
- 計測用トグル `SABORI_PROBE=0` を追加（`run_improvement_probe` 冒頭の早期 return に `probe_enabled_` ガード）。bench `bench_probe.py`（最適化36問×5シード、SAT 除外、勾配と同条件）。
- 結果: **probe_on vs probe_off = net −23（24–47）、5 シード中 4 で off 勝ち・1 引分・probe_on は無勝利**。勾配ヒント（−13）より明確な足枷。
- §7 を JP/EN とも更新: 勾配ヒント（平均で負け・ポートフォリオ向き）と improvement probe（より明確に有害）を分離。まとめ表の最適化行・結論バレットに probe を追記。

**設計判断・理由**
- **§7 の最適化レイヤは勾配＋probe の両方が net マイナス、probe の方が悪い**。`probe_fail_limit_` を既定で正に（probe ON に）している現状は最適化問題で損。**既定オフが妥当**（structural 既定（§4）と同じ見直し候補）。
- temporal（第1基準）の発覚と同じ教訓: **コードに別機構が同居しているとき、トグルが片方しか覆っていないと未計測の機構を見落とす**。§7 は「勾配＋probe」一括説明だったが、トグルは勾配だけだった。
- 向きの頑健性: 30秒 wall-clock 判定の run ブレはあるが、4/5 シード一貫＋probe_on 無勝利で、向きは確か。
