## 2026-02-13 Model SoA → AoS 変換

**実施内容**
- Model クラスの変数データを SoA (Structure of Arrays) から AoS (Array of Structures) に変換
- 7つの個別ベクタ (`mins_`, `maxs_`, `sizes_`, `initial_ranges_`, `support_values_`, `is_defined_var_`, `last_saved_level_`) を `std::vector<VarData> var_data_` に統一
- `VarData` 構造体を定義（48bytes、1キャッシュラインに収まる）
- `mins()/maxs()/sizes()` の配列返却メソッドを削除し、個別アクセサ `var_min()/var_max()/var_size()` に統一
- `var_data()` mutable アクセサを追加（`Variable::sync_soa()` 用）
- テストコードを新APIに対応

**設計判断・理由**
- ホットパスのアクセスパターン分析で、同一変数の min+max を同時に読むケースがほぼ全て（`select_variable`, `process_queue`, `int_lin_eq` propagation, `rewind_to`）
- AoS にすることで同一変数のデータが同一キャッシュラインに乗り、ベクタのデータポインタ load も1回で済む
- `mins()/maxs()/sizes()` の配列返却は呼び出し元がテストと `Variable::sync_soa()` のみだったため削除可能と判断

- ベンチマーク比較（`is` 問題での fails 数計測）は未実施

## 2026-02-13 08:25〜08:40 int_lin_le verify error 修正

**実施内容**
- cgt 問題で発生する `constraint verify error: int_lin_le` の調査・修正
- 当初の計画では `process_queue()` の SetMin/SetMax ケースのみが原因と想定していたが、デバッグ計装を追加して調査した結果、**真の原因は `Model::prepare_propagation()` の SoA 同期タイミング**であることを発見
- 2つの修正を適用:
  1. **`src/core/model.cpp`**: `prepare_propagation()` で `sync_from_domains()` を制約の `prepare_propagation()` ループの**前**にも呼ぶよう変更（presolve Phase 1 がドメインを直接変更した後、SoA が stale のまま制約が初期化されていた）
  2. **`src/core/solver.cpp`**: `process_queue()` の SetMin/SetMax ケースで、`update.value`（要求値）ではなく `model.var_min()`/`model.var_max()`（実際の値）を使い、両方向のバウンド変化をディスパッチするよう修正

**設計判断・理由**
- `sync_from_domains()` を前後で2回呼ぶのはやや冗長だが、`prepare_propagation()` 内で制約がドメインを変更する可能性もあるため、安全のために後にも残した
- process_queue の修正は RemoveValue ケースと同じパターンに統一（整合性のため）

- lazy bounds (`vd.min` が実際のドメイン最小値と一致しない）の扱いは、他の制約でも潜在的な問題になりうる

## 2026-02-13 DisjunctiveConstraint 非strict版対応 + dirty フラグ廃止

**実施内容**
- `DisjunctiveConstraint` に `bool strict` パラメータを追加し、`fzn_disjunctive`（非strict）に対応
  - `is_satisfied`, `overload_check`, `propagate_timetabling`, `presolve`, `check_initial_consistency` で `d_min==0` のタスクを skip
- FlatZinc パーサ（`model.cpp`）に `fzn_disjunctive` 登録を追加
- MiniZinc predicate `fzn_disjunctive.mzn` を新規作成、`redefinitions.mzn` に include 追加
- テスト `zero_duration.fzn` / `.expected` を追加（d=0 タスクが自由に重なることを検証）
- `docs/CONSTRAINTS.md` に `disjunctive` 行を追加
- dirty フラグによるソート済み配列キャッシュを廃止し、`propagate()` で毎回 `update_sorted()` を実行するように変更
  - `dirty_`, `TrailEntry`, `trail_` を削除
  - `save_trail_if_needed` / `rewind_to` を空実装に

**設計判断・理由**
- 非strict: MiniZinc 標準の `disjunctive` は `d[i]=0` のタスクを無視するセマンティクス。strict フラグで切り替えることで既存コードへの影響を最小化
- dirty フラグ廃止: 動作確認のための簡素化。毎回ソートするため trail 不要
