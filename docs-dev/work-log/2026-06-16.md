**設計判断・理由**
- 「変更前」は親コミット全体ビルドではなく HEAD 上の差分のみ revert で作成。
  親(98f11eb)は数ヶ月前で global.hpp 分割等の大量差分があり、全体 revert では
  トレイル化を分離できない（A/B が別物比較になる）ため。
- 公平性: 「変更前」はホットパスの O(N) ループ復元に加え、on_instantiate のカウンタ維持
  (push_back/mark_constraint_dirty/decrement) も除去。維持コストを before に負わせると
  最適化を過大評価するため、純差分（ループ削減 − 維持コスト）を測る形にした。
- 実行は minizinc 経由でなく事前コンパイル済み FZN の直接実行。flatten ノイズを排し
  定数倍差をクリーンに測る。compiled FZN は redefinitions 適用済みでサポート制約のみなので安全。
- TIMEOUT 問題は both=制限秒で構造的に同値 → timing 信号ゼロ。完走問題のみの集計を別途追加。

**計測結果**
- 健全性: 112/112 で before/after の obj/status 完全一致（カウンタ=ループ結果が等価で探索木不変を実証）。
- 全体: overall speedup x1.000（before 1055.18s / after 1054.69s、after faster 58 / slower 54 = ノイズ）。
- 完走のみ50問: x0.992（after slower 18 / faster 12）。完走>1s の20問: x0.991（slower 13 / faster 6）。
- 最重量級でも効果なし: fox-geese-corn 360万 fails→x1.00、hitori 240万→x1.02、
  freepizza 150万 fails(14s 完走)→x0.973（after が遅い）。
