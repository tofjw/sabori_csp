## 2026-02-27 14:06 Phase 4.3 完了: vars_ をベースクラスから削除

**実施内容**

- Phase 4 Step 4.3: `vars_` (vector<shared_ptr<Variable>>) をベースクラス `Constraint` から完全除去
  - コンストラクタ: `Constraint(vector<VariablePtr>)` → `Constraint(vector<size_t> var_ids)`
  - `set_variables()` → `set_var_ids()` に改名
  - `var_ptrs()` アクセサ削除、`update_var_ids()` ヘルパー削除
  - `extract_var_ids()` フリー関数を追加（VariablePtr→ID変換ヘルパー）
  - `init_watches()` を簡略化（デフォルト w1_=0, w2_=1 のみ設定、refine_watches が後で再配置）
- 全制約サブクラスの対応（約30ファイル）:
  - comparison.cpp (9制約): `Constraint({x, y})` → `Constraint(extract_var_ids({x, y}))`
  - arithmetic.cpp: 同パターン
  - logical.cpp: ArrayBoolAnd/Or/BoolClause のラムダを ID ベクタ構築に変更、BoolClause::on_last_uninstantiated を var_ids_ref() + map lookup に変換
  - global.hpp: 16クラスにプライベート `vars_` メンバーを追加
  - global/*.cpp: 各コンストラクタで `vars_(std::move(vars))` 追加、`update_var_ids()` → `var_ids_ = extract_var_ids(vars_)` に変換
  - array_int_maximum/minimum, int_element/int_element_monotonic: vars_ 参照を ID ベース比較に変換
- テスト修正: Test 21 "Constraint watches initialization" を新しい init_watches() 動作に合わせて更新

**設計判断・理由**

- vars_ を各サブクラスに移動する設計: presolve で直接ドメイン操作（var->remove() 等）が必要なため VariablePtr は制約ごとに保持。ベースクラスからは除去してメモリ節約
