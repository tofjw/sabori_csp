**設計判断・理由**
- bump_activity の `contains + 1/domain_size`: ドメインが小さい（確定に近い）変数ほど強く bump。矛盾の原因変数を絞り込む効果
- cumulative の2段構成: 局所（矛盾原因の近傍）と大局（リソース消費の重み）を併用。局所のみだと全体的なリソース配分が見えず、大局のみだとシグナルが散る
- is_satisfied 追加: verify_solution が nullopt スキップしていた制約を検出可能に
- bench スクリプトのプロセス残留: subprocess.run はタイムアウト時に子プロセスツリーを kill しない
