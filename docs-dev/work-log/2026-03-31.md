**実施内容**
- IntTimesConstraint の propagation バグ修正
  - `int_times(x, x, z)` で x と y が同一変数の場合、z 確定時に x への逆伝播が行われなかった
  - z=0 → x=0、z>0 → x^2=z のフィルタ、z<0 → 即矛盾を追加
  - bounds-only ドメイン対応: √z の整数根を計算し bounds 制限（O(1)）、sparse set は for_each_value
  - __int128 で overflow 対策（Codex レビュー指摘）
  - bounds-only で root がドメイン外の矛盾検出（Codex レビュー指摘）

**設計判断・理由**
- IntTimes x==y バグ: 2WL で x と y が同一変数の場合、z 確定→x 逆伝播のパスが既存の分岐条件では到達不能だった
- Cumulative 50/50: 局所のみだと全体のリソース配分が見えず、全体のみだとシグナルが散る。50/50 が最もバランス良い
- ArrayVarIntElement revert: ベースラインの「矛盾に直接関与した変数にだけ集中 bump」が VSIDS の学習効果として最も正しい原則。tightness は焦点がぼやける
- BoolClause revert: デフォルトの全確定変数均等配分で十分
