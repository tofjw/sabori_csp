## 2026-02-15 bool_lin_eq / bool_lin_le パラメータ名参照対応

**実施内容**
- `bool_lin_eq` / `bool_lin_le` が FlatZinc のパラメータ名参照（例: `bool_lin_eq(X_INTRODUCED_123_, X_INTRODUCED_121_, X_INTRODUCED_90_)`）に対応していなかった問題を修正
- 第1引数（係数配列）: `resolve_int_array()` を使用するよう変更
- 第2引数（変数配列）: `resolve_var_array()` + `get_var_by_name()` を使用するよう変更
- 第3引数（合計値/上限値）: FlatZinc 仕様では `var int` が許容されるため、変数の場合は線形式に組み込む変換を追加
  - `sum(c[i]*x[i]) = y` → `sum(c[i]*x[i]) + (-1)*y = 0`
  - `sum(c[i]*x[i]) <= y` → `sum(c[i]*x[i]) + (-1)*y <= 0`

**設計判断・理由**
- 第1・第2引数は `int_lin_eq` と同じ `resolve_int_array` / `resolve_var_array` パターンを適用
- 第3引数は `int_lin_eq` と異なり FlatZinc 仕様上 `var int` を許容するため、変数の場合は係数 -1 で線形式に追加して RHS=0 とする方式を採用
- 当初計画では第3引数は `par int` 前提だったが、gt-sort の FlatZinc で `var 0..30` の変数が渡されるケースを発見し対応を拡張

## 2026-02-15 IntElementConstraint bounds propagation 実装

**実施内容**
- `IntElementConstraint` に `on_set_min`/`on_set_max` コールバックを実装し、探索中の増分的な bounds propagation を追加
  - Forward propagation: index bounds → result bounds（Sparse Table による O(1) range min/max）
  - Reverse propagation: result bounds → index bounds（prefix/suffix 単調配列上の二分探索）
- コンストラクタで Sparse Table（O(n log n) 前処理）を構築
- `range_min(lo, hi)` / `range_max(lo, hi)` ヘルパーを実装
- cgt 問題で segfault を検出・修正
- テスト 6 + 5 = 11 セクション追加（71 → 95 assertions）

**設計判断・理由**
- Sparse Table を選択: forward propagation で厳密な区間 min/max が O(1) で得られる。メモリは O(n log n) だが定数配列のため1回構築で済む
- Stateless 維持: 全データ構造は定数配列から事前計算。trail/rewind 不要
- `enqueue_set_min/max` API 使用: プロジェクト規約通り
