## 2026-03-03 presolve 変更検出の軽量化

**実施内容**
- `PresolveResult` enum (`Contradiction`, `Unchanged`, `Changed`) を `constraint.hpp` に追加
- `Constraint::presolve()` の戻り値を `bool` → `PresolveResult` に変更（純粋仮想関数）
- 全38箇所の presolve 実装を変換（ヘッダ5ファイル、ソース24ファイル）
- `Solver::presolve` の固定点ループからドメインサイズのスナップショット比較を除去し、`PresolveResult` を直接使用するように変更
- テストファイル（`test_constraints.cpp`, `test_global_constraints.cpp`）のアサーションを更新

**設計判断・理由**
- No-op 制約（circuit, int_lin_le, int_lin_ne, int_lin_le_imp）→ `Unchanged` を返す。スナップショットの完全な無駄を排除
- Bounds 制約（int_lt, int_le, int_max, int_min 等）→ 既存のガード条件（`if (x_max > y_max)`）を利用して `changed` を追跡。追加コスト O(1)
- フィルタリング制約（all_different, table, int_eq 等）→ 除去ループ内で `changed = true` を設定。追加コスト無し
- ヘルパー委譲制約（int_abs, diffn, array_var_int_element）→ ヘルパーの戻り値が bool のまま変更不可（on_set_min 等からも呼ばれるため）。呼出し前後のドメインサイズスナップショットで変更検出（元の Solver レベルのスナップショットより軽量：対象変数のみ）
- 保守的 `Changed` は固定点ループの無限ループを引き起こすため使用禁止（初期実装で発見・修正）

**気づいた課題・TODO**

- IntTimesConstraint::presolve は propagate_bounds のインライン展開で正確な changed 追跡を実現済み
