## 2026-02-26 Activity VSIDS化・探索多様化・ベンチマークスクリプト改善

**実施内容**

- Activity 計算方式を VSIDS 風に変更
  - `activity[v] += activity_inc` 方式に変更
  - overflow（>10000）時に全体を max=100 にスケーリング、`activity_inc` をリセット
  - リスタート時: `activity_inc /= activity_decay`
  - コメントアウトされていた `bump_activity` 呼び出し（6箇所）をすべて有効化
- コミュニティ構造が弱い場合の bump_activity 無効化
  - modularity < 0.3 または最大クラスタが2番目の5倍超のとき無効化
  - `-v` で判定結果を表示
  - ※ユーザーにより一時的に常時有効（`return` で早期リターン）に変更済み
- リスタート時の探索多様化（`-c` なしでも有効に）
  - `select_restart_pivot()` メソッドに抽出
  - `-c` 有効時: 従来のコミュニティローテーション
  - `-c` 無効時: var_order_ を5グループに均等分割してローテーション、グループ内 MRV+activity で起点変数を選択
  - accap 問題で `-c` なしでも途中解が出るようになったことを確認
- ベンチマークスクリプト改善（bench_compare.py, bench_compare_2024.py）
  - TIMEOUT カウントを「解なし（obj=None）」のみに変更
  - 使用したデータファイル名を表示に追加（コンソール・HTML両方）

**設計判断・理由**

- VSIDS 風 activity: 明示的な decay ループを不要にし、increment のスケーリングで相対的な減衰を実現
- 探索多様化: コミュニティ分析はオーバーヘッドがあるため、`-c` なしでも簡易的なグループローテーションで同等の効果を得る
- TIMEOUT カウント: 途中解がある TIMEOUT は「完全な失敗」ではないため、解なし TIMEOUT のみカウントする方が実態に合う
