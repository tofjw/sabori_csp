  self-subsumption が唯一の有効技術と特定

  (protein/cg2023/solbat の決定論フィンガープリント完全一致、gate 差分は
  全てタイミングジッタ、soundness=0)
- **recursive minimization 実装**(MiniSat litRedundant、SABORI_LEARN_MINIMIZE):
  below_buf を pos 付き struct 化、locate+メモ化再帰で理由チェーンが
  基底(UIP/kept/root)に帰着する冗長リテラルを除去。因果非循環性で健全
  (amaze SOL、ctest 202/202、節は実際に短縮 removed 93k/60s)
- **しかし探索は大幅悪化**: minimize ON で solbat UNSAT 126s→未証明、
  cg2023 OPTIMAL 6340→TIMEOUT 12273 → **既定 OFF で確定**

  **このソルバーは LCG なしで好成績を出すよう探索側を作り込んであり、

  「探索と喧嘩させない統合・出し分け」が筋
- compute_reason は重複削減のクリーンアップとして採用価値あり(軌道中立)

**up-front dispatch 特徴量の候補（codex 提案 + 自分の整理）**

静的(presolve 後):
- explainable_weight: 説明可能制約族(bool_clause/reified比較/int_lin_*/
  one_hot)の incidence 割合。未対応グローバル(cumulative/disjunctive/
  circuit/all_different 主体)はペナルティ
- bool_reif_density: bool変数・reified制約・bool節・one_hot を決定変数あたり。
  高いと LCG 有利
- linear_tightness: 線形制約の正規化スラック。タイトな線形系は LCG 有利
- objective_coupling: 目的の次数、目的(連動補助変数含む)を含む制約数、
  **root 伝播による目的ドメイン縮小量(raw span より良い)**
- domain_vocabulary_fit: bool/小整数/bounds-only vs 穴あき大ドメイン の割合。
  穴が多いと decision-demote(語彙限界)で LCG が効きにくい
- global_vs_linear_ratio: 未対応グローバル重み高 + reification 低 → LCG 過負荷
- グラフ特徴: 変数あたり制約数、VIG 密度、community modularity。密な
  reified/linear は学習有利、分解可能な局所構造は restart/再降下探索有利

実行時(最初の N リスタート):
- explained_conflict_ratio、学習節長の中央値/短節率、nogood 有用度
  (prune+domain / check)、fallback 汚染率
- primal 進捗: 初解到達時間、リスタートあたり改善数、目的デルタ EMA、
  probe の SAT/UNSAT/UNKNOWN 比
- 探索撹乱: 学習前後の activity 順位相関、depth(浅い+高 explanation は
  LCG 有利 / 深い+cap 多発は base 有利)

検証: **leave-one-year-out**(第2インスタンス holdout は易しいインスタンス
過適合で auto-LCG が失敗した教訓)。粗い単調ルールを少自由度で。

## 2026-06-14 (午後) LCG dispatch 撤退判定 〜 submit 構成の対フィールド検証

**実施内容**
- LCG up-front dispatch の特徴量計測基盤を構築（/tmp から scripts/lcg_dispatch/ へ移設）
  - 118 nontie 問題（LEARN+ 63 / LEARN- 55）に対し 23 静的特徴を抽出（explainable_weight,
    bool_reif_density, linear_frac, global_vs_linear, mean_hole, VIG密度/modularity 等）
  - objective_coupling（root 伝播の目的ドメイン縮小量）を solver.cpp init_search に verbose 計測追加して取得
  - 分析: 信号を持つ特徴は linear_frac(AUC 0.337) のみ、他は AUC≈0.5。objective_coupling も無情報(0.518)
  - LOYO 検証: linear_frac 閾値ルール 0.593 vs 多数決 0.534（+6pp, 14年中5年で負け）
  - regret 評価: always-base 216 / always-L 158 / dispatch(LOYO) 151 → dispatch は always-L を +7(4%)しか改善せず
  → dispatch 撤退判定。up-front dispatch は特徴で予測不能
- submit 構成の3者比較（main / branch base / branch lcg, 2022-2025, 60s, minizinc 経由）
  - base > main（非LCG改修が改善）、lcg は解く力最強だが易問オーバーヘッド + 退行あり
- 対フィールド検証（base/lcg vs CP-SAT / Gecode, Challenge流スコア）
  - 対 CP-SAT(8t): -L スコア -1.12、取り返し0/取り逃し3
  - 対 Gecode(1t): -L スコア -2.88、取り返し0/取り逃し4。base は Gecode に 46勝17敗
  - → -L は対フィールドで点を生まず微減
- 大インスタンス（最大インスタンス選択）/ CP-SAT incomplete 領域の比較
  - CP-SAT 8t: sabori 完敗（base 0勝29負、10-160x 悪い目的値）
  - CP-SAT 1t: 逆転して互角（base 23勝20負）→ 大インスタンスの劣勢は CP-SAT の8並列がほぼ全て
  - base vs lcg: 17/16 → -L は incomplete 領域でも無益

**設計判断・理由**
- 撤退条件（頑健な相関なし/LOYO汎化せず）を満たしたため dispatch を打ち切り、-L なし(base)で submit を推奨
- accuracy でなく regret（誤判定で失う質）で評価 — 高インパクト事例の取りこぼしを重視
- 公平性のため CP-SAT を 1スレッドでも再実行 — 8並列が gap を誇張していたことが判明
- submit を base 推奨とする3根拠: (1)対フィールドで -L は点を生まず微減 (2)非LCGで競争力=新規性/貢献
  (3)健全性リスク回避（lcg-soundness-incident）
