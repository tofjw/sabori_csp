## 2026-02-04 継続作業 - Domain::remove() の根本的なバグ修正

**実施内容**
- hitori問題実行時の "bad optional access" ランタイムエラーをデバッグ
- 根本原因を特定：`Domain::remove()` が singleton から値を削除して空ドメインを作成していた
- `Domain::remove()` のセマンティクスを変更：
  - 存在しない値の削除: `false` → `true`（変更なしは成功）
  - 削除すると空になる場合: 削除せず `false` を返す（fail）
- 制約の `propagate()` メソッドで `remove()` の戻り値をチェックするよう修正：
  - `IntEqConstraint`, `IntNeConstraint`, `IntLtConstraint`, `IntLeConstraint`
  - `IntEqReifConstraint`, `IntNeReifConstraint`
- `Solver::run_search()` に防御的な空ドメインチェックを追加（二重の安全策）
- テストを修正：新しい `remove()` のセマンティクスに合わせた期待値、singleton 削除失敗テスト追加
- 全104テストがパス、hitori問題が解けることを確認

**設計判断・理由**
- 従来：`remove()` は値を削除して空になっても成功を返し、呼び出し側が `empty()` をチェックする設計
- 新設計：`remove()` が空になる削除を拒否し失敗を返す → 矛盾を即座に検出可能
- これにより、空ドメインの変数が存在する状態を防止
