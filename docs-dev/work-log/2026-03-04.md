**実施内容**
- dichotomic_search（B&B前フェーズ）を廃止し、improvement probe（解発見後プローブ方式）に置き換え
  - `solve_optimize()` から dichotomic_search 呼び出しを削除（init_search → search_with_restart_optimize 直結に）
  - `dichotomic_search()` メソッド本体（~195行）と solver.hpp の宣言を削除
  - `search_with_restart_optimize()` の SAT ブロック内に improvement probe を追加
    - 改善解発見後、root バックトラック＆目的変数縮小の直後に実行
    - ~5% 改善ターゲット（`range / 20`）で軽量プローブ（fail limit = 5）
    - SAT → best 更新＆永続縮小、UNSAT → bound tightening（`obj >= target + 1`）、UNKNOWN → そのまま通常 B&B に戻る
    - decision_trail_ への仮定記録を維持（nogood 正当性保証）
- `probe_fail_limit_` デフォルトを 256 → 5 に変更（solver.hpp, main.cpp）
- ヘルプテキスト・docstring 更新

**設計判断・理由**
- B&B 前フェーズの廃止: 最初の解が見つかる前に二分探索しても時間浪費。解発見後に挑む方が確実
- UNSAT 時の bound tightening: プローブが `obj <= target` の UNSAT を証明した場合、`obj >= target + 1` で永続的に下界を引き上げ（情報を捨てない）
- fail limit = 5, 改善幅 5%: ユーザー調整済み。UNKNOWN なら即通常 B&B に戻るので無駄が少ない
