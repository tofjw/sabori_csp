## 2026-02-16 探索多様性の改善

**実施内容**
- `select_variable` にランダムタイブレーキング（reservoir sampling）を導入
  - 同スコア（同 domain_size かつ同 activity）の候補から均一ランダムに1つ選択
  - `tie_count` をカウントし `rng_() % tie_count == 0` で置換する標準的な reservoir sampling
- 値順シャッフルは検討したが、NG の効きが悪くなる懸念から撤回
- NG 学習時の activity 更新ループで `break;` が入っていたバグを修正（全リテラルに activity が加算されるように）

**設計判断・理由**
- 変数選択のタイブレーキング: バックトラック時に `var_order_` の相対順序が復元されないため、スキャン順による決定的なタイブレーキングでは探索多様性が不足。reservoir sampling により O(1) 追加コストでランダム化
- 値順シャッフル撤回: NG は特定の (var, val) 組み合わせを禁止するため、値の試行順が毎回変わると同じ枝の早期枝刈りが効きにくくなる

  - `presolve()` の先頭でタイムライン・CP 配列をリセットする 3 行を追加

**設計判断・理由**
- バグの根本原因: `presolve()` がソルバーの固定点ループで複数回呼ばれる際、前回のタイムライン (`timeline_`, `cp_lo_`, `cp_hi_`) が残存していた。2 回目の呼び出しで、1 回目にセットした自身の古い CP ビットと衝突し偽の UNSAT が発生

- テストの `.expected` はソルバー実行結果から生成（探索順依存のため）

## 2026-02-16 Adaptive Restart 実装

**実施内容**
- Adaptive Restart（NG prune-driven outer ceiling control）を実装
  - 旧: `conflict_limit_multiplier_`（固定1.1）による指数成長
  - 新: inner/outer の2層構造。inner はべき乗成長（`*= inner_ratio_`）、outer は cycle 終了時に prune_delta と max_depth の変化に基づいて動的に grow/shrink
- outer 調整ロジック: prune が発生 **かつ** depth が伸びた場合のみ shrink、それ以外は grow
- パラメータ: `inner_ratio_=1.01`, `initial_outer_ceiling_=4.0`, `outer_min_=3.0`, `outer_max_=10000.0`, `grow_factor=1.2`, `shrink_factor=0.99`
- 最適化版では解改善時に outer をリセット

**設計判断・理由**
- 等間隔分割（10個、100個、200個）を試したが、最終的にべき乗成長に戻した（inner_ratio_=1.01）
- shrink 条件を「prune > 0 かつ depth 成長」に限定：prune が起きていても depth が停滞していれば deeper search が必要と判断

**気づいた課題・TODO**
- mondoku/atsp 両方で `depth_grew=0` が続き shrink がほぼ発動しない → shrink 条件の見直しが必要かもしれない
- outer は基本的に grow 方向のみで安定

**ベンチマーク結果**
- mondoku: 1.6s（最適解証明、restarts=2336、outer→14.3）
- atsp: 11.4s（最適解証明、restarts=1609、outer→20.6）
