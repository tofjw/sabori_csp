**設計判断・理由**
- **「測って晒す」方針**: 信頼狙い（CP/SAT 界隈）では未計測の独自性主張は逆効果。各機能を反証可能な A/B にかけ、効かないものも正直に出す方が信頼が積む。
- **記事の概念的背骨**: 計測群が「LCG は論理で無駄探索を演繹的に止める／sabori は同じ矛盾情報を主に activity に流して傾向で止める」という一線に収束。NoGood 分解（activity 寄与 +18 ＞ 枝刈り +9）がこれを直接裏付ける。タイトル「LCG と何が違うのか」への芯を食った答え。
- **「効く土台 vs 効かない精緻化」**: 効くのは activity を育てる土台（NoGood・mix_p）とモデル縮約（one-hot）。activity が先に決着するため、その上に乗せた精緻化（Bloom タイブレーク・構造特化・勾配）は出番が回らない、という統一的説明。
- **勾配＝値選択軸**: activity（変数選択）とは別軸。「得意不得意があること自体は欠点でない（並列アンサンブル前提なら多様性が資産）」という注を追加。
- **先行研究を隠さず固有性を絞る**: mix_p の核は Xia & Yap 2018 の貢献と明記し novelty を「混合比連続軸/EMA/per-restart」に限定。隠すより「先行を踏まえてこう振った」の方が界隈で信頼される。負け報告（コミュニティ分析・Bloom）は Liang & Ganesh 2015 で説明できる既知現象に格上げ＝むしろ強くなる。記事の締め「if there's an established name, I'd like to know」に自分で答えを埋める形。
- **成功の捉え方**: 記事（+SAT→CP 追試の研究価値）が Challenge 結果から独立した「保証された下限」。CP-SAT に迫る速度（過去ログ）なら強い placement は現実的な上振れ。下限が固く上も射程内、という構図。残るレバーは配信（HN/Reddit/MiniZinc Discourse 等）とテーゼの頑健性。
- **多シードの必要性**: 単一シードでは構造特化が「base に 9–6 で負け」と出たが、5 シードで差消滅（符号反転）。狭サンプルの勝敗はシードで入れ替わる罠を再確認。timing でなく目的値判定でも net 値は run 間でブレる（向きは安定）。
- **Bloom はソフト着地**: 「撤去」と断ぜず「指紋インフラ `var_ng_bloom` は残し、タイブレーク以外の用途を探すか撤去」。NG→activity が主経路だからタイブレークに出番が来ない、と機構的に説明。
- **main に -C を入れない**: feature/ng2 の conflict learning は既定オフ（-C ゲート）でベンチ未使用のため、記事の数字は main の既定挙動を正しく記述。cherry-pick 時に member 宣言の context 衝突が出たが `conflict_learning_` 行を落として解消。
