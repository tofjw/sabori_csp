## 2026-02-17 Bounds-Only Domain 実装

**実施内容**
- レンジが 10,000 を超える変数を bounds-only モードで管理する機能を実装
- sparse/dense 配列を確保せず、min/max + removed_values_ ベクタで完全性を保証
- atsp 問題で 6.3GB 使用していた Domain メモリを大幅に削減

**設計判断・理由**
- 閾値 10,000: remove の線形サーチコストと sparse set のメモリコストのトレードオフ
- removed_values_ のバックトラック: VarTrailEntry に old_removed_count を追加し、truncate で O(1) 復元
- bounds-only 変数は bisect 強制: values() が O(range) なので enumerate は非実用的
- values_ref() は bounds-only で assert 失敗: 呼び出し元を全て bounds-only 分岐で保護
