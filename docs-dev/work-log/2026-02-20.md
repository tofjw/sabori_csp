## 2026-02-20 element制約のindex変数をbisect対象から除外

## 2026-02-20 NoGoodブルームフィルタによる変数選択タイブレーク

**実施内容**
- NoGood ブルームフィルタによる変数選択タイブレークを実装
  - 128-bit ブルームフィルタ (`Bloom128` struct) を `model.hpp` に追加
  - 変数ごとに「どの NoGood が紐づいているか」をブルームで保持 (`var_ng_bloom_`)
  - 探索パス上で選択済み変数のブルームを OR で蓄積 (`ng_usage_bloom_`)
  - `select_variable` で domain_size + activity が同値のとき、bloom overlap (popcount) でタイブレーク
  - Trail に `ng_usage_bloom` を保存/復元し、backtrack で正しく戻る
  - リスタート時に `rebuild_var_ng_blooms()` で一括再構築（add_nogood 時ではなくリスタート時に反映）
  - NoGood に通し番号 `id` を付与、Fibonacci hashing で 2 ビット/NG を設定

**設計判断・理由**
- 128-bit (uint64_t x2) を採用: 64-bit では NG 数が増えるとすぐ飽和、256-bit はキャッシュ効率低下とのバランス
- `add_nogood` で Model を触らず、リスタート時に一括再構築: add_nogood に Model* を渡す変更を避け、1リスタート遅れは実用上問題なし
- Trail に bloom を保存: backtrack 時に OR の逆演算が不可能なため、push 前の状態を保存して復元
