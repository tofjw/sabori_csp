## 2026-03-21 int_div backward propagation 強化

**実施内容**
- `propagate_bounds` の y backward propagation を追加
  - z が確定時、`x/(z+1) < |y| ≤ x/z` の公式で y bounds を計算
  - x が正負に跨る場合は正側・負側の y 範囲を個別に計算し outer bounds（和集合の外接）を使用
  - レビューで発見した conflicting bounds バグ（false UNSAT）を修正
- `presolve` に 2 変数確定時のフィルタリングを追加
  - x,z 確定 → y ドメインから `x/y ≠ z` の値を除去（collect-then-remove でイテレータ安全）
  - y,z 確定 → x ドメインを `compute_x_range` で bounds tightening
  - これにより presolve 後に 1 変数のみ未確定でも 2WL に依存せず解決できる
- `abs_y_max` の使用は分析の結果 tight で正しいことが判明（前回レビューコメントの誤り）
- テスト 3 件追加: `y_backward_straddle`（x 正負跨り）、`presolve_xz_fixed`（verify error 検出）、`presolve_yz_fixed`

**設計判断・理由**
- y backward を `propagate_bounds`（探索中）と `presolve` の両方に追加した理由: presolve は 2 変数確定ケースの exact filter、propagate_bounds は bounds のみだが探索中に繰り返し呼ばれるため効果的
- x 正負跨りでの outer bounds: `set_min/set_max` では gap を表現できないため、安全側（広め）の bounds を使用
- presolve の y フィルタは `to_remove` ベクタで collect-then-remove: Domain のイテレータが remove で無効化される問題を回避
