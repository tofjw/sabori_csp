## 2026-02-28 Phase 4 完了: 制約内部の VariablePtr メンバ除去

**実施内容**
- Step 4.1 (前セッション完了済み): `variables()` pure virtual メソッド削除
- Step 4.2 完了: comparison (9クラス), arithmetic (3クラス) から VariablePtr メンバ除去
- Step 4.3 完了: logical 制約 (ArrayBoolAnd, ArrayBoolOr, BoolClause) から `vars_`, `r_`, `pos_`, `neg_`, `var_ptr_to_idx_` 除去
- Step 4.4 完了: global 制約全20クラスから `vars_`, `index_var_`, `result_var_`, `m_`, `x_`, `b_`, `index_`, `array_`, `result_`, `var_ptr_to_idx_` 除去
- Step 4.5 完了: `is_satisfied()` 整理 — 基底クラスに `is_satisfied(const Model&)` の汎用実装を追加（`on_final_instantiate` 経由）
- テスト修正: TableConstraint の `on_final_instantiate` → `is_satisfied` 無限再帰を解消、IntElementConstraint / IntLeReifConstraint の初期整合性テストを `presolve()` ベースに変更

**設計判断・理由**
- コンストラクタ引数は `VariablePtr` のまま維持（FznBuildContext が渡すため）。内部メンバのみ除去
- presolve 内では `model.variable(id)->` で Variable* を取得してドメイン直接操作（presolve では許可）
- propagation コールバックでは `model.enqueue_*` API のみ使用（既存のまま）
- ArrayBoolAnd/Or のコンストラクタ watch 初期化を簡略化（prepare_propagation で再初期化されるため）
- BoolClauseConstraint の `can_satisfy(size_t)`, `is_satisfied_by(size_t)`, `get_var(size_t)` (Model引数なし版) を削除、Model引数版のみ残す
- DisjunctiveConstraint のタスクヘルパー (`task_start`, `task_dur` 等) に `const Model&` 引数を追加
- DiffnConstraint の `propagate_pairwise_direct` に `Model&` 引数を追加
- 基底 `is_satisfied(const Model&)` は全変数確定時に `on_final_instantiate` を呼ぶ実装（`const_cast` 使用）
